# jianghu-init v6 配置说明

本文档说明 `version: 'v6'` 的配置写法、组件规范、解析流程与各层职责边界。

---

## 1. 设计目标与三层架构

v6 采用**声明 → 解析 → 拼接**的半编译架构，三层各司其职：

| 层 | 文件 | 职责 |
|---|---|---|
| **Schema（声明层）** | `page-template-json/example/*.js` | 只写页面声明，不写任何 if/else 业务逻辑 |
| **Analyzer（解析层）** | `lib/json/v6/parse_schema.js` | 解析 Schema，注入默认值，将每个节点补全 `resolvedComponent` 和 `resolvedProps` |
| **Generator（拼接层）** | `page-template-json/jh-page/jh-page-v6.njk.html` | 遍历节点树，输出 `<vue组件名 :prop="val">`，**不做任何业务判断** |

**核心约束**：业务渲染逻辑（如 table 怎么展示操作按钮、分页如何切换）全部住在 **Vue 组件内部**，NJK 只负责最后一公里的字符串拼接。

---

## 2. 配置入口条件

满足以下条件时触发 v6 解析流程：

- `version === 'v6'`
- 存在 `page` 字段，且至少有 `pageContent` 或 `dataSource` 之一

入口代码：`lib/json/mixin.js` → `lib/json/v6/parse_schema.js`

---

## 3. Schema 顶层结构

```js
{
  version: 'v6',
  pageType: 'jh-page',          // jh-page | jh-mobile-page | jh-component
  page: {
    id: 'pageId',
    name: '页面名称',
    type: 'list',               // list | detail | custom
  },
  pageContent: [ /* 布局+业务节点树，见第 4 节 */ ],
  actionContent: [ /* 抽屉/弹窗节点，见第 5 节 */ ],
  dataSource: { /* 数据源声明，见第 6 节 */ },
  resourceList: [],             // 资源列表，透传
  includeList: [],              // 引用列表，透传
  common: { /* 自定义 data/methods 等，见第 7 节 */ },
}
```

---

## 4. pageContent：布局节点 + 业务节点树

`pageContent` 是一棵由**布局 Primitive** 和**业务 Block** 组成的声明树。
Analyzer 对整棵树做深度优先遍历，给每个节点补全 `resolvedComponent`（Vue 组件标签名）和 `resolvedProps`（解析后的最终 props）。
NJK 消费这两个字段输出 Vue 模板，**不读原始 `component` 字段**。

### 4.1 节点通用结构

```js
{
  component: 'ComponentName',   // 见下方清单
  props: { /* 直接对应 Vue 组件的 props，键名一致 */ },
  children: [ /* 子节点，仅布局 Primitive 有效 */ ],
}
```

**`props` 与 Vue 组件 props 一一对应**，Analyzer 只做默认值注入和 `xxxRef` 引用解析，不做结构变换。

---

### 4.2 布局 Primitive 清单

布局 Primitive 只负责 flex/grid 布局，不含任何业务逻辑，**必须**有 `children`。

#### `VStack`（垂直堆叠）

```js
{
  component: 'VStack',
  props: {
    gap: 12,                    // 子元素间距，单位 px，默认 0
    align: 'start',             // align-items：start | center | end | stretch，默认 stretch
    justify: 'start',           // justify-content：start | center | end | between，默认 start
    padding: '0 16px',          // CSS padding 简写，默认无
  },
  children: [ /* ... */ ],
}
```

生成 Vue 标签：`<jh-vstack>`

---

#### `HStack`（水平排列）

```js
{
  component: 'HStack',
  props: {
    gap: 8,
    align: 'center',            // 默认 center
    justify: 'start',           // 默认 start
    wrap: true,                 // 是否允许换行，默认 false
    padding: '',
  },
  children: [ /* ... */ ],
}
```

生成 Vue 标签：`<jh-hstack>`

---

#### `Box`（自由盒子）

```js
{
  component: 'Box',
  props: {
    padding: '16px',
    margin: '0',
    width: '100%',
    style: { /* 任意额外 CSS */ },
  },
  children: [ /* ... */ ],
}
```

生成 Vue 标签：`<jh-box>`

---

#### `Grid`（栅格）

```js
{
  component: 'Grid',
  props: {
    cols: 3,                    // 列数，默认 1
    gap: 12,                    // 间距，单位 px，默认 0
    colsMd: 2,                  // md 断点列数（可选）
    colsSm: 1,                  // sm 断点列数（可选）
  },
  children: [ /* ... */ ],
}
```

生成 Vue 标签：`<jh-grid>`

---

### 4.3 业务 Block 清单

业务 Block 是有具体渲染逻辑的组件，props 与 Vue 组件定义完全对应。**不需要** `children`。

#### `PageHeader`（页头 + 搜索区）

```js
{
  component: 'PageHeader',
  props: {
    title: '页面标题',           // 不填则取 page.name
    searchFieldList: [
      {
        key: 'fieldKey',        // 绑定字段名
        label: '显示标签',
        op: 'like',             // like | eq | in，影响搜索逻辑
        type: 'text',           // text | select | date | dateRange
        options: [],            // type=select 时提供选项
      },
    ],
  },
}
```

生成 Vue 标签：`<jh-page-header>`

---

#### `TableBlock`（数据表格）

```js
{
  component: 'TableBlock',
  props: {
    columns: [
      {
        key: 'fieldKey',
        label: '列标题',
        width: 120,             // 列宽，px，可选
        sortable: true,         // 是否可排序，默认 true
        formatter: '',          // 自定义渲染模板片段（HTML 字符串），可选
      },
    ],
    rowActionList: [
      {
        label: '编辑',
        intent: 'update',       // update | delete | custom
        id: 'actionId',         // intent=custom 时必填
        confirm: false,         // 是否需要二次确认，默认 false
      },
    ],
    headActionList: [
      {
        label: '新增',
        intent: 'create',       // create | custom
        id: 'actionId',
      },
    ],
    serverPagination: true,     // 服务端分页，默认 false
    pageSize: 50,               // 每页条数，默认 50
    selectable: false,          // 是否显示复选框，默认 false
    primaryKey: 'id',           // 行唯一键，不填则取 dataSource.primaryKey
  },
}
```

生成 Vue 标签：`<jh-table>`

## 5. actionContent：抽屉/弹窗节点

`actionContent` 用于声明抽屉、弹窗等 overlay 层，**不放入** `pageContent`。

### 5.1 节点通用结构

```js
{
  component: 'CreateFormBlock',   // 见下方清单
  key: 'create',                  // 抽屉唯一标识，影响 data/methods 命名
  props: { /* 同样直接对应 Vue props */ },
}
```

### 5.2 表单抽屉

#### `CreateFormBlock`（新增抽屉）

```js
{
  component: 'CreateFormBlock',
  key: 'create',
  props: {
    title: '新增',
    fields: [
      {
        key: 'fieldKey',
        label: '字段标签',
        type: 'text',           // text | select | date | textarea
        required: false,
        readonly: false,
        options: [],            // type=select 时提供
        autoId: {               // 自动生成 ID（可选）
          prefix: 'C',
          start: 10001,
        },
      },
    ],
  },
}
```

生成 Vue 标签：`<jh-create-drawer>`

---

#### `UpdateFormBlock`（编辑抽屉）

```js
{
  component: 'UpdateFormBlock',
  key: 'update',
  props: {
    title: '编辑',
    fields: [ /* 同 CreateFormBlock.fields */ ],
  },

```

生成 Vue 标签：`<jh-update-drawer>`

## 8. Analyzer 职责（parse_schema.js）

Analyzer 对整棵 `pageContent` 和 `actionContent` 树做**深度优先遍历**，对每个节点：

1. **`resolvedComponent`**：将 `component` 名转换为 Vue 组件标签名（kebab-case，加 `jh-` 前缀）

   | component 名 | resolvedComponent |
   |---|---|
   | `VStack` | `jh-vstack` |
   | `HStack` | `jh-hstack` |
   | `Box` | `jh-box` |
   | `Grid` | `jh-grid` |
   | `PageHeader` | `jh-page-header` |
   | `TableBlock` | `jh-table` |
   | `CreateFormBlock` | `jh-create-drawer` |
   | `UpdateFormBlock` | `jh-update-drawer` |

2. **`resolvedProps`**：处理 `props` 中的引用语法（`xxxRef` → 从 schema 中按路径取值），注入各组件默认值，**不做结构变换**，结果直接绑定到 Vue 组件 props。

3. **`children`**：递归对子节点做同样处理。

4. **提取 `standardConfig`**：从树中找到 `PageHeader`、`TableBlock` 等节点，提取 `blocks` 用于 Analyzer 侧的逻辑（如 feature flags），同时保留完整节点树供 NJK 消费。

Analyzer **不构造** `headActionList`、`rowActionList`、`formItemList` 等 v5 专属结构；这些结构住在 Vue 组件内部。

## 9. NJK Generator 职责（jh-page-v6.njk.html）

NJK 分两部分：

### 9.1 模板区（`<script type="text/html">`）

只遍历节点树，输出 Vue 组件标签 + props 绑定，**不出现任何业务判断**：

```
<%- for node in standard.pageContent %>
  <{node.resolvedComponent}
    <%- for key, val in node.resolvedProps %>  :{key}='{val | json}'
    <%- endfor %>
    <%- for attr, varName in node.resolvedBindings %>  {attr}="{varName}"
    <%- endfor %>
  >
    <%- 布局容器：递归渲染 node.children %>
  </{node.resolvedComponent}>
<%- endfor %>
```

新增组件类型：只改 `parse_schema.js` 的注册表，NJK 模板区不动。

### 9.2 脚本区（`<script type="module">`）

提供**标准 CRUD Runtime**，包含：

- `defaultUiActionConfig`：action → 原子方法名数组的调度表
- `doUiAction`：调度层，查 `common.doUiAction`（页面覆盖）再查 `defaultUiActionConfig`，按数组顺序 `await` 每个原子方法
- 各原子方法：每个只做一件事，可在 `common.methods` 中单独 override

**不出现**：直接判断 `uiActionId === 'xxx'` 的硬编码分发。

## 10. doUiAction 原子方法链设计

v6 继承 v4 的原子化方法链模式：**`doUiAction` 只调度，业务逻辑在原子方法里**。

### 标准调度表（`defaultUiActionConfig`）

| actionId | 原子方法链（按序执行） |
|---|---|
| `getTableData` | `prepareTableParams` → `fetchTableData` → `formatTableData` |
| `startCreateItem` | `prepareCreateFormData` → `openCreateDrawer` |
| `createItem` | `prepareCreateValidate` → `prepareDoCreateItem` → `doCreateItem` → `closeCreateDrawer` → `getTableData` |
| `startUpdateItem` | `prepareUpdateFormData` → `openUpdateDrawer` |
| `updateItem` | `prepareUpdateValidate` → `prepareDoUpdateItem` → `doUpdateItem` → `closeUpdateDrawer` → `getTableData` |
| `deleteItem` | `prepareDeleteFormData` → `confirmDeleteItemDialog` → `prepareDoDeleteItem` → `doDeleteItem` → `getTableData` |

### 扩展与 Override 规则

**替换整条链**（在 `common.doUiAction` 里覆盖同名 key）：

```js
common: {
  doUiAction: {
    // 完全替换 deleteItem 的调用链
    deleteItem: [
      'prepareDeleteFormData',
      'confirmDeleteItemDialog',
      'myCustomDeleteCheck',   // 自定义原子方法
      'doDeleteItem',
      'getTableData',
    ],
  },
}
```

**替换单个原子方法**（在 `common.methods` 里 override 同名方法）：

```js
common: {
  methods: {
    // 只替换 formatTableData，其余链条不动
    formatTableData() {
      this.tableData = this.tableDataFromBackend.map(row => {
        row.classTypeName = { '普通班': '普通', '重点班': '重点' }[row.classType] || row.classType;
        return row;
      });
    },
    // 自定义原子方法，供 doUiAction 链中引用
    async myCustomDeleteCheck() {
      if (this.deleteItem.locked) throw new Error('该记录已锁定，不可删除');
    },
  },
}
```

## 10. 完整示例（classV6.js）

```js
const content = {
  version: 'v6',
  pageType: 'jh-page',
  page: { id: 'exampleClassV6', name: '班级页面-v6', type: 'list' },

  pageContent: [
    {
      component: 'VStack',
      props: { gap: 12 },
      children: [
        {
          component: 'PageHeader',
          props: {
            searchFieldList: [
              { key: 'className', label: '班级名称', op: 'like', type: 'text' },
              { key: 'classType', label: '班级类型', op: 'eq', type: 'select', options: ['普通班', '重点班', '特长班'] },
            ],
          },
        },
        {
          component: 'TableBlock',
          props: {
            columns: [
              { key: 'classId',      label: '班级ID',   width: 80 },
              { key: 'className',    label: '班级名称' },
              { key: 'classType',    label: '班级类型' },
              { key: 'classBalance', label: '班费' },
              { key: 'remarks',      label: '备注' },
            ],
            rowActionList: [
              { label: '编辑', intent: 'update' },
              { label: '删除', intent: 'delete', confirm: true },
            ],
            headActionList: [
              { label: '新增', intent: 'create' },
            ],
            serverPagination: true,
            pageSize: 50,
            selectable: true,
          },
        },
      ],
    },
  ],

  actionContent: [
    {
      component: 'CreateFormBlock',
      key: 'create',
      props: {
        title: '新增班级',
        fields: [
          { key: 'classId',   label: '班级ID',   type: 'text',   readonly: true, autoId: { prefix: 'C', start: 10001 } },
          { key: 'className', label: '班级名称', type: 'text',   required: true },
          { key: 'classType', label: '班级类型', type: 'select', required: true, options: ['普通班', '重点班', '特长班'] },
          { key: 'remarks',   label: '备注',     type: 'text' },
        ],
      },
    },
    {
      component: 'UpdateFormBlock',
      key: 'update',
      props: {
        title: '编辑班级',
        fields: [
          { key: 'classId',   label: '班级ID',   type: 'text',   readonly: true },
          { key: 'className', label: '班级名称', type: 'text',   required: true },
          { key: 'classType', label: '班级类型', type: 'select', required: true, options: ['普通班', '重点班', '特长班'] },
          { key: 'remarks',   label: '备注',     type: 'text' },
        ],
      },
    },
  ],

  dataSource: {
    table: 'example_class',
    primaryKey: 'id',
    listResource:   'selectItemList',
    createResource: 'insertItem',
    updateResource: 'updateItem',
    deleteResource: 'deleteItem',
  },

  resourceList: [
    { actionId: 'selectItemList', resourceType: 'sql', resourceData: { table: 'example_class', operation: 'select' } },
    { actionId: 'insertItem',     resourceType: 'sql', resourceData: { table: 'example_class', operation: 'jhInsert' } },
    { actionId: 'updateItem',     resourceType: 'sql', resourceData: { table: 'example_class', operation: 'jhUpdate' } },
    { actionId: 'deleteItem',     resourceType: 'sql', resourceData: { table: 'example_class', operation: 'jhDelete' } },
  ],

  common: {
    data: {
      constantObj: {
        classType: ['普通班', '重点班', '特长班'],
      },
      tableSelected: [],
    },
    dataExpression: {
      isMobile: 'window.innerWidth < 500',
    },
    created: async function() {
      await this.doUiAction('getTableData');
    },
    doUiAction: {
      batchConsole: ['batchConsole'],
    },
    methods: {},
  },

  includeList: [],
};

module.exports = content;
```
